Encampments & Social Determinants of Health (San Diego)

A tract-level, privacy-preserving data workflow linking pre-aggregated encampment counts with American Community Survey (ACS) socioeconomic indicators for San Diego County — built as a reproducible pipeline that produces analysis-ready tables for descriptives and modeling.

Overview

This project creates an analysis-ready, tract-level dataset that pairs tract-aggregated encampment counts with tract-level ACS indicators. The intent is to support descriptive analysis and modeling of relationships between encampments and social determinants of health (SDoH), while maintaining responsible handling of sensitive information.

  • Geography: San Diego County census tracts
  • Unit of analysis: tract-level counts + tract-level ACS indicators
  • Join key: GEOID (stable tract identifier)

Privacy and data handling

Encampment data are already aggregated by census tract to protect anonymity. This project does not use point locations or individual-level records.

  • No point geometries / no individual locations
  • No personally identifiable information (PII)
  • All analysis occurs at an anonymized geographic unit (tract)

Workflow

1) Ingest

  • Pull ACS 5-year tract estimates (demographic, housing, economic)
  • Normalize variable names and schemas
  • Construct derived indicators where needed

2) Transform

  • Standardize tract identifiers (GEOID)
  • Join ACS indicators to tract-aggregated encampment counts
  • Run missingness + coverage checks

3) Analysis

  • Descriptives (distributions, scaling, outliers)
  • Correlation / exploratory relationships
  • Modeling appropriate for aggregated counts

How to run

From the repository root, run scripts in order (ingest → transform → analysis).

# Example (adjust names to match your repo)
Rscript src/ingest/01_ingest_acs.R
Rscript src/transform/02_join_encampments_acs.R
Rscript src/analysis/03_descriptives_models.R

If you use renv / Make targets / an R project file, update this section to match.

Outputs

  • Tract-level joined dataset (analysis-ready)
  • Summary tables / descriptives
  • Model outputs (fit summaries + diagnostics)

Outputs are intentionally kept small and reviewable for portfolio purposes.

Tools

  • R
  • tidycensus, dplyr, tidyr
  • sf (only when tract geometry is needed)

Note: This project uses tract-level aggregation to support responsible analysis of a sensitive topic. The emphasis is on reproducible workflows and privacy-aware design.
