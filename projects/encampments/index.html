Encampments & Social Determinants of Health (San Diego)

A tract-level data engineering and analysis workflow linking aggregated homeless encampment counts with American Community Survey (ACS) socioeconomic indicators for San Diego County. Built to be reproducible and privacy-safe.

Overview

This project demonstrates how to create an analysis-ready, tract-level dataset that pairs pre-aggregated encampment counts with tract-level ACS indicators. The goal is to support descriptive analysis and statistical modeling of relationships between encampments and social determinants of health (SDOH), while maintaining responsible handling of sensitive information.

Scope
  • Geography: Census tracts (San Diego County)
  • Unit of analysis: tract-level aggregated counts + tract-level ACS indicators
  • Methods: reproducible ingestion, variable construction, joins, descriptives, modeling

Privacy and data handling

Encampment data are already aggregated by census tract to protect privacy and anonymity. This project does not use point locations or individual-level records.

Key constraints
  • No point geometries / no individual locations
  • No personally identifiable information (PII)
  • All analysis occurs at an anonymized geographic unit (tract)

Workflow

1) Ingest

  • Pull ACS 5-year tract estimates using tidycensus
  • Normalize variable names and schemas
  • Aggregate age/demographic bands where needed

2) Transform

  • Standardize tract identifiers (GEOID)
  • Join ACS indicators with pre-aggregated encampment counts
  • Check missingness and coverage

3) Analysis

  • Tract-level descriptive summaries
  • Explore correlations and relationships
  • Regression modeling appropriate for aggregated count data

How to run

From the repository root, run the scripts in order (ingest → transform → analysis).

# Example (adjust script names to match your repo)
Rscript src/ingest/01_ingest_acs.R
Rscript src/transform/02_join_encampments_acs.R
Rscript src/analysis/03_descriptives_models.R

Note: If your repo uses an R project file, renv, or Make targets, swap this section to match that workflow.

Outputs

  • Tract-level joined dataset (analysis-ready)
  • Summary tables / descriptives
  • Model outputs (coefficients, diagnostics, fit summaries)

Outputs are intentionally kept small and reviewable for portfolio purposes.

Tools
